# 요구사항 명세서 (Requirements Specification)

**프로젝트명**: CollaBoard  
**버전**: 1.0.0  
**작성일**: 2025-01-13  
**목표**: 2026 웹 프론트엔드 개발자 포트폴리오

---

## 1. 프로젝트 개요

### 1.1 목적
실시간 협업이 가능한 칸반 보드 애플리케이션으로, Trello와 유사하지만 다음 차별점에 중점을 둠:
- **복잡한 상태 관리 구현 능력 증명** (보드-리스트-카드 3단 구조)
- **실시간 멀티유저 동기화 구현 능력**
- **고성능 최적화 능력** (100+ 카드 환경에서 60fps 유지)
- **접근성 및 테스트 코드 작성 능력**

### 1.2 타겟 사용자
- 소규모 팀 (2-10명)
- 프로젝트 관리가 필요한 개발자/디자이너
- 칸반 방식의 워크플로우 선호자

### 1.3 핵심 가치 제안
1. **빠른 성능**: 대규모 보드에서도 끊김 없는 드래그앤드롭
2. **실시간 협업**: 다른 사용자의 변경사항을 즉시 확인
3. **간결한 UI**: 복잡한 기능 없이 핵심 기능에 집중
4. **접근성**: 키보드만으로 모든 작업 가능

## 2. 기능 요구사항 (Functional Requirements)

### 2.1 MVP 기능

#### FR-1. 보드 관리
- **FR-1.1** 사용자는 새로운 보드를 생성할 수 있다.
  - 입력 항목: 보드 제목 (필수, 1-100자)
  - 배경색 선택 (선택, 6가지 프리셋)
- **FR-1.2** 사용자는 보드 목록을 조회할 수 있다.
  - 최신 순 정렬
  - 최근 접속한 보드 강조 표시
- **FR-1.3** 사용자는 보드를 수정할 수 있다.
  - 제목, 배경색 변경 가능
- **FR-1.4** 사용자는 보드를 삭제할 수 있다.
  - 삭제 전 확인 모달 표시
  - 삭제 시 하위 리스트/카드 모두 삭제

#### FR-2. 리스트 관리
- **FR-2.1** 사용자는 보드 내에서 리스트를 생성할 수 있다.
  - 입력 항목: 리스트 제목 (필수, 1-50자)
  - 보드 맨 오른쪽에 추가됨
- **FR-2.2** 사용자는 리스트 제목을 수정할 수 있다.
  - 인라인 편집 (클릭 시 입력 모드)
- **FR-2.3** 사용자는 리스트를 삭제할 수 있다.
  - 확인 없이 즉시 삭제 (Undo 제공)
- **FR-2.4** 사용자는 리스트 순서를 변경할 수 있다.
  - 드래그앤드롭으로 수평 이동
  - 키보드 단축키 지원 (Alt + ←/→)

#### FR-3. 카드 관리
- **FR-3.1** 사용자는 리스트 내에서 카드를 생성할 수 있다.
  - 입력 항목: 카드 제목 (필수, 1-200자)
  - 리스트 맨 아래에 추가됨
- **FR-3.2** 사용자는 카드를 다른 리스트로 이동할 수 있다.
  - 드래그앤드롭으로 이동
  - 리스트 내 순서 변경 가능
  - 키보드 단축키 지원 (Alt + ↑/↓/←/→)
- **FR-3.3** 사용자는 카드 상세 정보를 조회/수정할 수 있다.
  - 제목 (필수)
  - 설명 (선택, 마크다운 지원)
  - 라벨 (선택, 다중 선택 가능)
  - 담당자 (선택, 단일 선택)
- **FR-3.4** 사용자는 카드를 삭제할 수 있다.
  - 확인 없이 즉시 삭제 (Undo 제공)

#### FR-4. 실시간 협업
- **FR-4.1** 다른 사용자의 변경사항이 실시간으로 반영된다.
  - 보드/리스트/카드 생성/수정/삭제/이동
  - 최대 지연시간: 500ms
- **FR-4.2** 현재 접속 중인 사용자 목록을 표시한다.
  - 사용자 아바타 (우측 상단)
  - 최대 5명까지 표시, 초과 시 "+N" 표시
- **FR-4.3** 동시 편집 충돌을 방지한다.
  - 낙관적 업데이트 (Optimistic Update)
  - 충돌 시 마지막 업데이트 우선 (Last Write Wins)
  - 실패 시 에러 토스트 + 롤백

#### FR-5. 인증
- **FR-5.1** 사용자는 이메일로 회원가입/로그인할 수 있다.
  - 이메일 형식 검증
  - 비밀번호: 8자 이상, 영문+숫자 조합
  - JWT 토큰 기반 인증
- **FR-5.2** 게스트 모드를 지원한다.
  - 로그인 없이 사용 가능
  - 로컬 스토리지에 데이터 저장
  - "회원 전환" 기능 제공

### 2.2 2순위 기능 (Post-MVP)
- **FR-6** 카드 코멘트
- **FR-7** 파일 첨부 (이미지/문서, S3 업로드)
- **FR-8** 활동 로그 (타임라인)
- **FR-9** 검색/필터 (제목, 라벨, 담당자)
- **FR-10** 다크모드

## 3. 비기능 요구사항 (Non-Functional Requirements)

### 3.1 성능 (Performance)
- **NFR-1** 100개 이상의 카드가 있는 보드에서 60fps 유지
- **NFR-2** 드래그앤드롭 레이턴시 16ms 이하 (1 프레임)
- **NFR-3** 초기 로딩 시간 3초 이하 (3G 환경 기준)
- **NFR-4** Time to Interactive (TTI) 5초 이하
- **NFR-5** WebSocket 재연결 시간 2초 이하

### 3.2 확장성 (Scalability)
- **NFR-6** 단일 보드 내 최대 리스트: 50개
- **NFR-7** 단일 리스트 내 최대 카드: 500개
- **NFR-8** 동시 접속 사용자: 최대 50명 (단일 보드 기준)

### 3.3 접근성 (Accessibility)
- **NFR-9** WCAG 2.1 Level AA 준수
- **NFR-10** 키보드 네비게이션 100% 지원
- **NFR-11** 스크린 리더 호환 (ARIA 속성)
- **NFR-12** 색상 대비 비율 4.5:1 이상

### 3.4 테스트 (Testing)
- **NFR-13** 유닛 테스트 커버리지 80% 이상
- **NFR-14** E2E 테스트: 주요 사용자 플로우 100% 커버
- **NFR-15** 모든 PR에 자동 테스트 실행 (CI)

### 3.5 보안 (Security)
- **NFR-16** JWT 토큰 만료 시간: 7일 (Refresh Token: 30일)
- **NFR-17** 비밀번호 bcrypt 해싱 (salt rounds: 10)
- **NFR-18** XSS 방지 (사용자 입력 이스케이프)
- **NFR-19** CORS 정책 적용 (허용 도메인만 접근)

### 3.6 브라우저 호환성
- **NFR-20** Chrome 최신 2개 버전
- **NFR-21** Firefox 최신 2개 버전
- **NFR-22** Safari 최신 2개 버전
- **NFR-23** Edge 최신 2개 버전
- **NFR-24** 모바일 반응형 지원 (최소 너비: 320px)

### 3.7 코드 품질
- **NFR-25** TypeScript strict mode 활성화
- **NFR-26** ESLint 규칙 0 에러, 0 경고
- **NFR-27** Prettier 자동 포맷팅 적용
- **NFR-28** 함수 복잡도 (Cyclomatic Complexity) 최대 10
- **NFR-29** 컴포넌트 파일 최대 300줄

## 4. 제약사항 (Constraints)

### 4.1 기술 제약
- React 18.3+ 사용 (Concurrent Features 활용)
- TypeScript 5.3+ (strict mode)
- 번들 크기 제한: 초기 로드 500KB 이하 (gzip)
- Node.js 20+ LTS 버전 사용

### 4.2 비용 제약
- 무료 티어 서비스 우선 사용
  - Vercel (프론트엔드)
  - Railway/Render (백엔드)
  - Upstash Redis (실시간 상태)
- S3 사용량: 월 5GB 이하 (2순위 기능)

### 4.3 일정 제약
- MVP 완성 목표: 8주
- 주차별 마일스톤 설정 (별도 문서 참조)

## 5. 우선순위 (Priority)

### Must Have (P0)
- 보드/리스트/카드 CRUD
- 드래그앤드롭
- 실시간 동기화
- 이메일 인증

### Should Have (P1)
- 카드 상세 (라벨, 담당자)
- 게스트 모드
- 키보드 네비게이션
- 80%+ 테스트 커버리지

### Nice to Have (P2)
- 코멘트
- 파일 첨부
- 활동 로그
- 검색/필터
- 다크모드

## 7. 위험 요소 및 대응 (Risks & Mitigation)

| 위험 | 영향도 | 확률 | 대응 전략 |
|------|--------|------|-----------|
| 실시간 동기화 구현 복잡도 | 높음 | 중 | Socket.io 공식 예제 학습, 단계별 구현 |

| 무료 티어 제한 초과 | 낮음 | 낮음 | 사용량 모니터링, 대체 서비스 사전 조사 |
| 테스트 커버리지 미달 | 중 | 중 | TDD 원칙 준수, 코드 리뷰 체크리스트 |
